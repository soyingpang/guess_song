# 華語詩歌估歌仔：團契版玩法計劃書

更新時間：2026-05-20 17:45 HKT

## 1. 目的

這個 app 目標是給教會團契現場使用，用詩歌作破冰、競賽和敬拜回憶遊戲。主持人在電腦後台控制流程，前台大螢幕給全場觀看，團友用手機加入答題。

這份文件是後續開發的玩法和產品規格基準。其他 AI 或開發者接手時，應先讀本文件，再讀 `docs/AI_HANDOFF.md` 和 `docs/UPDATE_LOG.md`。

## 2. 器材配置

現場會使用同一部電腦接兩個畫面：

- `MON1`：主持後台控制畫面。
- `MON2`：前台大螢幕畫面，給全場觀看。
- 後台和前台是在同一部電腦、同一個瀏覽器環境開兩個視窗。
- 聲音只應由前台大螢幕畫面播放。
- 後台只負責控制，不應發聲。
- 手機端只負責加入、答題、搶答和看分數，不播放詩歌。

團友約 10 人，用手機掃前台 QR code 加入。

## 3. 角色

### 3.1 主持人

主持人使用後台畫面，負責：

- 開前台視窗。
- 複製前台連結，給外地朋友開遠端前台。
- 使用固定房間，不需要每次遊戲重新開新房。
- 選擇遊戲環節。
- 自由切換三個環節，不限制每個環節題數。
- 設定播放秒數難度。
- 開始播放 / 停止播放。
- 開放搶答。
- 判定答中 / 未中。
- 開估。
- 下一題。
- 顯示排行榜或公布勝方。
- 重置分數，開始新一局但保留房間、QR、玩家和題庫。
- 複製玩家加入連結，作 QR 掃不到時的後備。
- 管理玩家和分組。

### 3.2 前台大螢幕

前台是全場主要畫面，負責：

- 顯示遊戲狀態。
- 顯示 QR code。
- 播放詩歌聲音；如題目使用本地授權影片檔，也可顯示影片畫面。
- 顯示倒數。
- 顯示搶答結果、主題搶唱題、答案和排行榜；四選一歌名只在手機端顯示，前台不顯示歌名選項。
- 遮好 YouTube 標題、縮圖文字或任何會露答案的資訊。
- 如使用本地 MP4 / 影片檔，影片內容本身也不應在未開估前露出歌名、字幕或明顯答案。
- 如不在主持同一部電腦，必須使用帶房間碼的前台連結連回主持後台。
- 前台預設就是有聲播放，不再顯示「啟用聲音」按鈕；若個別遠端瀏覽器仍阻擋有聲 autoplay，屬瀏覽器政策，需在該裝置用瀏覽器/播放器互動處理。
- 前台右下角必須一直保留玩家手機 QR。即使等待同步或未收到後台狀態，也要用固定房間生成玩家加入連結。

後台主持頁排版原則：
- 普通桌面寬度保留大型影片預覽，方便主持手動跳過或處理 YouTube 廣告。
- 超寬畫面才把影片與控制列並排，避免普通螢幕影片太細。
- 玩家、房間連結和題庫管理放在右側卡片側欄，主持主控區放左側。
- 狀態列用高可讀性的淺色紙卡，確保播放狀態和答案不會被背景吃掉。
- 後台不提供估歌作答，不顯示可點擊的四選一選項；玩家估歌只在手機端進行。
- 如果固定房間已被另一個後台佔用，這個後台必須鎖住出題、播放、開估等控制，避免手機和前台連到另一個後台而不同步。

### 3.3 手機玩家

玩家用手機：

- 掃 QR code 加入房間。
- 輸入名字。
- 選擇或顯示組別。
- 在四選一環節答題。
- 在搶答環節按搶答。
- 查看自己分數、排名和狀態。

手機端不播放詩歌。
四選一題必須在手機端顯示選項；如暫時未收到選項，應顯示同步提示。

## 4. 整體遊戲流程

1. 主持開後台。
2. 主持按「開前台」。
3. 前台視窗拖到 `MON2`。
4. 如有外地朋友要看前台，主持按「複製前台連結」傳給他們。
5. 外地朋友開前台後，前台會直接嘗試按主持指令有聲播放。
6. 前台顯示 QR code。
7. 團友用手機掃 QR code，輸入名字加入。
8. 主持確認玩家列表。
9. 主持設定或即場選擇：
   - 目前玩哪個環節。
   - 播放秒數：60 秒 / 30 秒 / 15 秒。
   - 分組設定：A 組 / B 組。
10. 主持按「下一題」開始第一題；系統會按目前秒數設定自動播放。
11. 主持可隨時切換環節、下一題、開估、重新搶答或顯示排行榜。
12. 遊戲沒有固定題數限制；完場由主持自行決定，按「公布勝方」或「排行榜」作結。
13. 如要再玩新一局，主持按「分數重置」清分數和題目狀態；同一個房間、QR、前台和已加入玩家全部保留。

建議流程順序，但不是限制：

1. 四選一選擇題。
2. 按時間搶答估歌。
3. 主題搶唱分組賽。

## 5. 播放時間和難度

後台需要提供清楚的播放秒數設定：

- `60 秒`：容易，適合熱身或較少聽詩歌的組員。
- `30 秒`：標準，建議預設。
- `15 秒`：困難，適合熟悉詩歌的組員或刺激環節。

規則：

- 播放秒數應由主持在後台設定。
- 播放秒數是主持即場控制項，可每題前自由更改。
- 播放起點也由主持在後台設定，可選 `由頭播` 或 `隨機中段`。
- 主持按「下一題」時，前台應即時按目前秒數設定自動播放。
- 若選 `隨機中段`，每題抽一次中段起點，同一題重播應沿用同一段。
- 前台播放時顯示倒數。
- 倒數完前台自動停止播放。
- 後台和手機只顯示狀態，不發聲。
- 主持按「開估」後，前台應自動播放全首詩歌；這個全首播放不受 60 / 30 / 15 秒限制。
- 開估全首播放應由歌曲開頭開始，不受 `隨機中段` 影響。

重要：現場需求是「只有前台有聲」。後續開發時要把播放控制集中到前台。

## 6. 環節一：四選一選擇題

### 6.1 玩法

- 系統隨機抽一首詩歌。
- 前台播放詩歌。
- 手機端顯示 4 個選項。
- 每位玩家每題只可答一次。
- 時間到後鎖定答案。
- 主持按「開估」後，前台顯示正確答案。

### 6.2 建議規則

- 答中：個人 `+1 分`。
- 答錯：`0 分`。
- 不扣分。
- 時間到仍未答：`0 分`。

### 6.3 畫面要求

後台：

- 顯示目前題號。
- 顯示已答人數 / 未答人數。
- 顯示答中人數。
- 顯示正確答案，只在主持需要時可見。

前台：

- 顯示題號和倒數。
- 顯示四個選項。
- 未開估前不顯示答案。
- 開估後顯示答案。

手機：

- 顯示四個大按鈕。
- 答完後顯示「已提交」。
- 不可重複作答。

## 7. 環節二：按時間搶答估歌

### 7.1 玩法

- 系統隨機抽一首詩歌。
- 前台播放詩歌和倒數。
- 主持按「開放搶答」後，手機端顯示可搶答狀態。
- 第一個按搶答的玩家取得答題權。
- 玩家現場口頭回答。
- 主持在後台按「答中」或「未中」。

### 7.2 建議規則

- 答中：個人 `+2 分`。
- 未中：`0 分`。
- 未中後主持可按「重新開放搶答」。
- 同一題可以設定最多搶答次數，但第一版可先不做限制，由主持控制。

### 7.3 主持流程

1. 載入題目。
2. 確認前台無廣告或已準備好。
3. 按「開始播放」。
4. 按「開放搶答」。
5. 第一位玩家搶答成功。
6. 玩家口頭回答。
7. 主持判定答中 / 未中。
8. 開估或重新開放搶答。
9. 下一題，或切去其他環節。

### 7.4 畫面要求

前台：

- 顯示「搶答開放」或「等待主持開放搶答」。
- 有玩家搶到時，大字顯示玩家名字。
- 開估後顯示答案。

手機：

- 顯示一個大搶答按鈕。
- 未開放時按鈕 disabled。
- 搶到時顯示「你搶到了，請回答」。
- 未搶到時顯示「已有人搶答」。

## 8. 環節三：主題搶唱分組賽

### 8.1 玩法

- 不播放 YouTube。
- 前台顯示一個大路主題 / 關鍵詞，例如「平安」、「恩典」、「愛」、「信」。
- 分 A 組和 B 組。
- 團友鬥快用手機搶答。
- 搶到的組要現場唱出一句切合該主題 / 關鍵詞的詩歌。
- 主持人工判定是否接受。

### 8.2 分組方式

建議第一版使用：

- 玩家加入時由系統自動平均分配 `A 組` 或 `B 組`；人數較少的一組優先，同人數時隨機。
- 後台可顯示每組人數。
- 後台可由主持手動調組。

### 8.3 建議規則

- 答中：該組 `+2 分`。
- 未中：`0 分`。
- 未中後主持可重新開放搶答。
- 同一首歌不應重複使用，但第一版可由主持人工記住。

是否計個人分：

- 第一版建議只計組分，不計個人分。
- 因為這環節重點是分組氣氛，而不是個人競賽。

### 8.4 主題庫

內置主題庫建議用非常大路、容易即場唱到的詞，避免太考驗專業詩歌知識：

```text
平安、恩典、愛、信、盼望、喜樂、感謝、讚美、敬拜、禱告、耶穌、天父、聖靈、十架、救恩、生命、光明、道路、真理、倚靠、同行、保守、安慰、醫治、得勝、祝福、赦免、榮耀、永恆、應許、回家、呼召
```

主持功能：

- 隨機抽主題。
- 手動輸入主題 / 關鍵詞。
- 跳過此主題。

### 8.5 畫面要求

後台：

- 顯示目前主題。
- 抽主題 / 手動輸入主題。
- 開放搶答。
- 答中 / 未中。
- 顯示 A 組 / B 組分數。

前台：

- 大字顯示目前主題。
- 顯示 A 組和 B 組分數。
- 顯示搶答成功的玩家和組別。

手機：

- 顯示目前主題。
- 顯示自己組別。
- 顯示大搶答按鈕。

## 9. 分數設計

第一版建議：

| 環節 | 計分 |
| --- | --- |
| 四選一 | 答中個人 +1 |
| 搶答估歌 | 答中個人 +2 |
| 主題搶唱 | 答中組別 +2 |

排行榜：

- 全場完結顯示個人 Top 5。
- 同時顯示 A 組 / B 組總分。
- 前台可以先顯示分組勝方全屏畫面，再由主持切到個人排行榜。

## 10. YouTube 廣告處理

YouTube 不能保證沒有廣告，所以 app 流程要容許主持處理廣告。

更專業和穩定的方案是不用 YouTube：把教會有權使用的音訊放入 `audio/`，或把有權使用的影片放入 `video/`，題庫填 `audioUrl` 指向 `./audio/song.mp3` 或 `./video/song.mp4`。`audioUrl` 是舊欄位名，現時代表「本地 / 已授權媒體 URL」；前台會自動按副檔名用 `<audio>` 或 `<video>` 播放，避免廣告和跨地區延遲。

建議流程：

1. 主持按「載入下一題」。
2. 前台載入 YouTube，但仍顯示遮罩。
3. 如果有廣告，主持可以先處理。
4. 主持按「正式播放」。
5. 倒數才正式開始。
6. 主持按「開估」後，前台公開答案並自動播放全首詩歌，方便全場確認和一起聽。

後台應有狀態：

- `預備中`
- `可開始`
- `播放中`
- `時間到`
- `開估，全首播放中`

## 11. 後台介面規劃

後台應改成現場控制台，避免被題庫佔太多畫面。

主要區域：

- 遊戲狀態。
- 現在環節。
- 題號。
- 播放秒數。
- 主要控制按鈕：
  - 開始遊戲
  - 載入下一題
  - 開始播放
  - 停止播放
  - 開放搶答
  - 答中
  - 未中
  - 開估
  - 下一題
  - 排行榜
  - 公布勝方
- 玩家列表。
- 玩家手動改組和移除離線玩家。
- 個人分數。
- 分組分數。

題庫區：

- 預設收起。
- 只在設定或維護時打開。

## 12. 前台介面規劃

前台要清楚、溫馨、適合教會團契，不應太像冷冰冰的考試系統。

最新設計方向：

- 使用深酒紅、燭光金、鼠尾草綠和暖白作主色。
- 最新方向不應是黑色風格；主調要是暖白紙面、淺木色、玫瑰木、鼠尾草綠和燭光金。
- 使用本地 SVG 素材，不依賴外部圖片服務。
- 背景可有淡淡教會窗格 / 樂譜線紋理，但不能干擾讀字。
- 更具體的視覺氣氛應是「都會團契的家」：城市窗景、暖燈、木桌、詩歌本、杯、植物，像大家下班後返到一個舒服地方一齊玩。
- 可用柔和燈串、布紋和邀請卡式 QR code 細節提升親切感，但不能影響主持操作或玩家掃碼。
- 花葉角落、柔和布邊和卡片內框可用來提升 lounge 感，但必須保持答案、題目、分數和 QR code 的可讀性。
- 紙卡 / 桌面活動卡質感可用於分數牌、QR 邀請卡、主持控制區和手機按鈕，令整體更像完整團契活動套件。
- 介面應以一張完整主視覺作核心，不應只靠簡單圖示或零碎圖案裝飾。
- 目前主視覺為 `assets/fellowship-main-visual-manhwa.png`：韓式漫畫 / webtoon 手繪感的都會客廳團契、暖燈、詩歌本、結他和城市夜景。
- YouTube 遮罩必須是實色或完整覆蓋，不可用半透明讓答案看穿。
- 前台和手機都要保留足夠大字、清楚按鈕和高對比，方便團契現場和投影使用。
- 可以保留深色文字和少量深酒紅作對比，但不要用大片黑底營造氣氛。

前台要顯示：

- 遊戲名稱。
- 現在環節。
- 題號。
- 倒數時間。
- QR code。
- 四選一狀態，但不顯示四個歌名選項，避免觀眾誤以為前台可以按。
- 搶答狀態。
- 主題搶唱的大字。
- 答案公開。
- 分組分數。
- 排行榜。
- 分組勝方公布畫面。

前台必須避免：

- 露出 YouTube 歌名。
- 露出 YouTube 頻道標題中含有歌名的文字。
- 半透明遮罩讓答案被看穿。

## 13. 手機介面規劃

手機畫面要根據環節自動變化。

加入前：

- 輸入名字。
- 不需要自選組別；系統會自動平均分 A/B 組。
- 加入遊戲。

環節一：

- 顯示四個選項。
- 答完後顯示已提交。

環節二：

- 顯示搶答按鈕。
- 顯示是否開放搶答。
- 搶到時顯示請回答。

環節三：

- 顯示目前主題。
- 顯示自己組別。
- 顯示搶答按鈕。

共通：

- 顯示自己分數。
- 顯示目前排名入口；排行榜放在彈窗，避免常駐佔用答題畫面。
- 顯示連線狀態。
- 可開咪對話，讓主持在後台聽到玩家答案。

開咪規則：

- 玩家主動按「開咪對話」並授權手機麥克風。
- 後台玩家列表顯示誰正在開咪。
- 主持可以按「收咪」中斷該玩家咪高峰。
- 手機斷線或重連時應自動關咪。
- 歌曲播放仍以前台大螢幕為主，手機咪高峰只作對話/回答用途。

## 14. 技術實作優先次序

### 第一階段：玩法骨架

1. 加自由場控遊戲狀態：等候、預備、播放中、時間到、開估、排行榜。
2. 加三個環節模式：`choice`、`buzzSong`、`wordSing`。
3. 加播放秒數設定：60 / 30 / 15。
4. 改成只有前台播放聲音。
5. 前台加倒數和自動停止播放。
6. 手機加入時由後台自動平均分 A/B 組。
7. 後台和前台顯示組分。
8. 加主題搶唱題庫。
9. 加主持手動顯示排行榜。
10. 加主持手動公布分組勝方。

### 第二階段：現場穩定性

1. QR code 改成本地生成，不依賴外部 QR API。（已完成）
2. 玩家斷線重連。（已完成）
3. 前台連線狀態提示。
4. 防止重複加入同名玩家。（已完成）

### 第三階段：題庫和美化

1. 每首歌加審核狀態。
2. 加經典程度 / 適合團契程度。
3. 題庫可按主題分包。
4. 前台 / 後台 / 手機端溫馨團契風格美化。（已完成第一輪）

6. 手機端按真機尺寸再微調。

## 15. 已確認方向

- 不限制每個環節玩幾多題。
- 不做系統自動完場。
- 主持可以在後台自由控制幾時玩哪個環節。
- 排行榜和分組勝方由主持按需要手動顯示。

## 16. 仍需用戶確認

以下仍需主持人確認：

1. 環節二答錯後是否一定可以重新開放搶答，還是由主持每次決定。
2. 環節三是否只計組分，不計個人分。
3. 是否需要主持可手動微調個人分數。
4. 手機端是否都要顯示分組勝方公布狀態，還是只保留大螢幕公布。

## 17. 建議預設場控方式

如果沒有再改，建議第一版採用：

- 開場先玩四選一熱身，主持想停就停。
- 中段切去搶答估歌，30 秒或 15 秒都可以即場改。
- 最後切去主題搶唱，分 A/B 組製造氣氛。
- 主持覺得時間差不多，就先按「公布勝方」，再按「排行榜」作結。
- 玩家加入時自己選 A/B 組。
- 最後先顯示分組勝方，再顯示個人 Top 5。
